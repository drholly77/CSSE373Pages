Changing a Project's Settings

Each time a new project is created, you must change its specific settings. You only need to do this once for each project you create. Failure to successfully change the settings will usually cause millions of compiler errors to be generated.

To change a project's settings, do the following:

From Visual Studio Solution Explorer, with your project open, right click L1 (or LX, depending on the project - See the screen shot to the right) and from the popup menu choose Properties, this will cause the "Property Pages" dialog to appear.

From the Property Pages dialog do the following:

  1. Choose All Configurations from the drop down box at the top of the dialog
  2. Select the C/C++ folder, and then select the General category
  3. Then on the right hand side of the dialog add the following path names to Additional Include Directories:
    ..\wrapper\include;..\components\include
  4. Next, select Precompiled Headers category  (Note: there is no screen shot for this step)
    • then on the right side select from the drop down box "Create/Use Precompiled Header":
      Not Using Precompiled Headers

 

To finish, click the OK button at the bottom of the Property Pages dialog.
