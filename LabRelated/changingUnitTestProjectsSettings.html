Changing a Unit Test Project's Settings

Each time a new project is created for unit testing, you must change its specific settings. You only need to do this once for each project you create. Failure to successfully change the settings will usually cause millions of compiler errors to be generated.

To change a project's settings, do the following:

From Visual Studio Solution Explorer, with your unit test project open do steps 1 and 2 listed in the screen shot to the right to bring up the Project Properties dialog

From the Property Pages dialog do the following 6 steps (Steps 1 - 3 in first screen shot, steps 4 - 6 in the second screen shot):

  1. Choose All Configurations from the drop down box at the top of the dialog
  2. Select the C/C++ folder, and then select the General category
  3. Then on the right hand side of the dialog add the following path names to Additional Include Directories:
    ;..\wrapper\include;..\components\include

    Note: be sure to include the two semicolons
