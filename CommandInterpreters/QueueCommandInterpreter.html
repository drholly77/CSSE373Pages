Queue Command Interpreter

Last Updated: 12/17/2015

Testing Harness - the Queue Command Interpreter serves as a harness that allows a software developer to exercise each of the Queue's exported operations by permitting the user to make explicit calls to these exported operations

Each do Operation:

  • Takes keyboard input from the user and then calls one of the Queue's exported operations
  • doClear calls clear
  • doTransferFrom calls transferFrom
  • doCopy calls operator =
  • doEnqueue calls enqueue
  • doDequeue calls dequeue
  • doReplaceFront calls replaceFront
  • doLength calls length
  • doFront calls front

Example: doLength

doLength works as follows - see implementation below:

  • allows the user to query the length of either q1 or q2
  • then to obtain a queue's length, it calls q1.length() or q2.length(), depending on the user's choice
  • then it displays a message to the user with the user selected queue's length

The other do operations work similarly
