5) Putting All the Operations Together - Seeing Design-By-Contract In Action

Everyone on the team together should work on this step

 

  1. On one team member's machine repeat Step 2 (Create a Main Program - above), but this time when asked for a name, give the name "Team #", where X is your #, assigned to you by Dr. Holly
    This naming convention will allow me to import all submissions for grading without name clashes
  2. In this new program combine all 7 operations into one program, which are: Read_Queue, Write_Queue, Rotate, Split_Into, Combine, Copy_Queue, and Main
  3. In the Main program insert a comment containing the names of all team members
  4. Be sure that the operations developed in Step 4 contain the required comment with the name of the author of the individual operation
  5. Work to get this combined program to verify and compile
  6. Modify Main so it makes suitable calls to the six other operations, i.e., do some traditional testing with concrete values
  7. Build a jar file and run your tests

Everyone on the team should be present to verify that this step is completed

  2. Upload the exported .json file to this assignment
  3. You can double check that you made a good export by having one of your other team members do an import of this .json file and then inspecting the imported combined program
