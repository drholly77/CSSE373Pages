2. ProjP2 Statement of Work

2.1 Primary Objectives

The primary objective of this lab is to provide you experience with:

  • Building a component directly in the programming language's raw data types, in this case with C++ Nodes & Pointers
  • This implementation of Sequence will require you to write recursive operations
  • Note: this implemenation will not maintain a balanced binary tree

 

2.2 Specifics

Steps:

  1. Start Visual Studio
  2. Open project L5Driver
  3. Implement each Sequence0 member functions - use the L5Driver command interpreter to test your implementation

Hints:

  • As a guide, use the nodes & pointers implementation of Queue found in Queue0.hpp of CppDevSp20/components/include/Queue
  • Implement all of Sequence0's stubbed out member functions and private helper functions
  • Many of your private helper member function implementations will look similar to Sequence3's implelementations
  • Look for the TODO comments in Sequence0.hpp
  • Reference the Sequence and BinaryTree specifications

Important Restrictions:

  1. Obey the correspondence - When implementing the member functions you must obey the correspondence internal contract
    That means:
    • assume that the correspondence holds when the member function is called
    • guarantee that the correspondence holds when the member function exits
  2. Use Design by Contract - No defensive programming of Sequence's operations
    • What this means is that the implementation of a Sequence operation must not contain code that checks to make sure that the Sequence operation was called correctly by the client
  3. No Cross Calling - None of Sequence's member functions are permitted to make calls to any of the other public Sequence member functions, i.e., no cross calling
    • All Sequence operations should directly manipulate Sequence's data members
  4. Calling Down - is encouraged (but not required) by creating private member functions that capture code that starts to appear in multiple different places in your Sequence0 implementation. Do this to avoid the duplication of code
  5. Recursion - You must used recursion for all private member functions
  6. About Making Copies -
    • When implementing Sequence0's operator = you must make a duplicate of the the binary tree which means you cannot just copy the pointer to the root of the tree and be done. You have to traverse the entire tree and while traversing you must construct a duplicate tree. While doing the traversal, you must call type T's operator equal in order to make a copy of the data stored in each node.
