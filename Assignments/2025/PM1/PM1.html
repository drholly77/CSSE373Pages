1.3 About Project PM1

  • It contains a Visual Studio 2022 project (see screen shot in Figure 3)
  • It can be opened with VS2022 by double clicking on the file PM1.sln
  • It will build correctly if you have followed the instructions in 1.1 - 1.2 (above)
  • PM1 contains .h files and driver.cpp files that you will work on to complete this part of the project
    The ToDos are in operation main, see (4.4) below
    • (1) CharacterCArray.h - creates an instance of CeramicArray; used to hold the puzzle word, e.g., the word "encyclopedia"; No ToDo here
    • (2) TextSet.h - creates an instance of component Set; used to hold the dictionary words; No ToDo here
    • (3) UnlabeledUndirectedGraph.h - creates instance of graph component; used to hold your graph data; No ToDo here
    • (4) driver.cpp - driver program that loads the program's data and executes the DFS on the graph
    • (4.1) initializeVertexLabelMap - operation that puts the puzzle word into a ceramic array variable; No ToDo here
    • (4.2) loadDictionaryWords - operation that loads the dictionary words from 'dataDictionary.dat' file (provided) into a variable of type TextSet; No ToDo here
    • (4.3) loadPuzzleData - operation that loads the puzzle data into your graph and puzzle word variables; No ToDo here
    • (4.4) main - main operation that declares variables and calls the operations listed above; ToDos are here

2. PM1 Statement of Work

Solve the Encyclopedia Search Puzzle (Figure 6)

  • As your starting place, use the driver.cpp found in "PM1 Starter Files" (see References above)
  • Modify PM1's driver.cpp by finding the ToDos and completing them
  • Your implementation must:
    1. Be in a Visual Studio project called PM1 and be stored in your repository
    2. The driver.cpp must use the operations provided that load the dictionary and graph data (see 4.1 through 4.2 in Figure 3)
    3. Use depth first search algorithm(s) search the puzzle for words of a specified length
    4. Must first output the found words by completing ToDo's 2 and 3 in driver.cpp's main operation
    5. Use the words loaded in from the file dataDictionary.dat to check the legitimacy of a computed word
    6. Be layered on UnlabeledUndirectedGraph0 implementation
    7. Leverage other already existing appropriate components from the component list whose implementations can be found in the folder: components\include - in the diagram to the right this is represented by the arrow to the Other Concepts oval
    8. By leverage (in #5 above) we mean, for example:
      • If you need searching or sorting, then find a component that already has that functionality, i.e., do not reimplement these algorithms
      • If you need a container to temporarily store things, then find a component that provides that functionality
      • Use the minimalization principle, i.e., attempt to find a component that provides the minimal functionality required
  • Recommended:
    • Test and debug with dataReact.dat, a smaller data set based on puzzle word react - see data files listed in Figure 2
      Graph diagram for react is in Figure 7
    • The instructor's solution when run with the command line arguments found in Figure 4 found the following words:
      "tea", "tet", "cat", "car", "ate", "aer", "eat", "ear", "ret", "rat"
    • Then test with dataEncyclopedia.dat (Figure 8)
      The instructor's solution working on dataEncyclopedia.dat and dataDictionary.dat found
      53 words of length 5
      34 words of length 6
      21 words of length 7
      7 words of length 8
    • Use checking components when in Debug Mode

3. Grading of PM1

  • Your driver.cpp from the folder PM1 will be compiled/built/tested
  • Your program will be tested/executed against a graph data file and word search length of the instructor's choosing
  • Your output, i.e., the words found by your search program, will be redirected to an output file and compared against the instructor's solution
  • Assessment will be based on the actual words found and the code found in driver.cpp
