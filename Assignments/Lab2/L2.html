1. Creating a unit test driver

Setup Part 1)

Create a VS2017 unit test project called: L2

  1. Navigate to the CSSE373 Moodle site - click on the Resource: Visual Studio Related Topics
  2. Follow the instructions in the two web pages under: Creating unit test projects
  3. Set Up the Wrapper Folder by the instructions in the web page: Adding .cpp Files to a Project

When you are finished with the 3 steps above your L2 unit test project should look like the screen shot to the right

Setup Part 2)

Create IntegerList.h

  1. Find the folder CppDevSp19\Queue1UnitTest and copy a file named IntegerQueue.h from that folder to the folder CppDevSp19\L2
  2. Rename IntegerQueue.h to IntegerList.h
  

Setup Part 3)

In Visual Studio, add the IntegerList.h file to the Header Files folder

  1. Right click the Header Files folder in the Solution
  2. Click "Add Existing Item" to add the IntegerList.h file to the Header Files solution folder

When you are finished with the 3 steps above your L2 unit test project should look like the screen shot to the right

Setup Part 4)

Edit the file IntegerList.h

  1. While in Visual Studio, double click the IntegerList.h file name found in the Header Files folder
  2. Edit this file by changing the following:
    • #include List\List1.hpp instead of Queue\Queue1
    • typedef IntegerList from List1 instead of IntegerQueue from Queue1
  

Setup Part 5)

Edit the top of the file unittest1.cpp by adding the following 2 #include lines

  1. #include "Wrapper.h"
  2. #include "IntegerList.h"

When you are finished, the file should look like unittest1.cpp file should look like the following:

   
#include "stdafx.h"
#include "CppUnitTest.h"
#include "Wrapper.h"
#include "IntegerList.h"

using namespace Microsoft::VisualStudio::CppUnitTestFramework;

namespace L2
{      
   TEST_CLASS(UnitTest1)
   {
   public:
      
      TEST_METHOD(TestMethod1)
      {
         // TODO: Your test code here
      }

   };
}

Setup Part 6)

Build your project

  1. Click from the menu bar: Build | Build Solution
  2. You should not have any build errors, if you do then one of the steps above was done incorrectly. You'll need to fix it before moving on

2. Create Test Cases For IntegerList

2.1 Requirements for constructing a unit test procedure

  • Each unit test that you create must have 8 sections as described in the reference materal: Constructing Unit Tests
  • It will help you to reference the examples of Queue unit tests which can be found in the file unittest1.cpp in the folder CppDevSp19\Queue1UnitTest

2.2 Create the following unit test procedures for IntegerList

  • Edit the unittest1.cpp file in your L2 project and create unit test procedures
  • Below is a list of 15 unit tests that you must create for this lab
  • Use the name as given in each of the 15 unit tests
  • Make each test use the provided incoming values
  • Make each unit test correctly verify that the operation performed its required function according to the operation's spec
UT01ConstructorV1: List1();
	outgoing: s1 = (<>,<>)
UT02ClearV1: s1.clear();
	incoming: s1 = (<>,<>)
	outgoing: s1 = (<>,<>)
UT03ClearV2: s1.clear();
	incoming: s1 = (<>,<33>)
	outgoing: s1 = (<>,<>)
UT04TransferFromV1: s1.transferFrom(s2);
	incoming: s1 = (<>,<>) and s2 = (<>,<44>)
	outgoing: s1 = (<>,<44>) and s2 = (<>,<>)
UT05AssignmentV1: s1 = s2;
	incoming: s1 = <> and s2 = (<>,<17>)
	outgoing: s1 = (<>,<17>) and s2 = (<>,<17>)
UT06AddRightFrontV1: s1.addRightFront(k);
	incoming: s1 = (<>,<21>) and k = 35
	outgoing: s1 = (<>,<35,21>) and k = 0
