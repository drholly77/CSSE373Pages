2. ProjP2 Statement of Work

2.1 Primary Objectives

The primary objective of this assignment is to provide you experience with:

  • Layering a component (e.g., Map) on an already existing component (e.g., BinaryTree)
  • An abstract math model for Binary Trees
  • Investigate formal methods concepts related recursion
  • Revisit binary search tree (BST) in an implementation of Map

Part 2 - Layering a Component

Open VS2019 and begin working on Map3.hpp by first finding comments that begin with Step X (where X is a number from 1 to 5). Start with Step 0 and work your way through to Step 5, carefully following the instructions of each Step.

Step 6 recommends an ordering for implementing and testing the operations of Map3

Run the driver by choosing from Debug from the Menu Bar, and then choosing either Start Debugging or Start Without Debugging

All Map3 specific operations must call a private recursive helper operation. These operations have been stubbed and specified in Map3.hpp. The table below indicates that the Standard Operations will use call throughs to BinaryTree component operations, while the Map3 Specific Operations call a corresponding recursive helper operation.

Exported Map Operation   Private Map Operation
the constructor doesn't call BinaryTree's constructor automatically called
the destructor doesn't call BinaryTree's destructor automatically called
transferFrom calls call through to BinaryTree
operator = calls call through to BinaryTree

add calls insertInTree (see supplied code in Map3.hpp)
size calls countNodes
Until you get this recursive countNodes version working, have your countNodes operation just call BinaryTree's size operation. And when writing your recursive countNodes operation, you are permitted to call BinaryTree's size to test for base case.
hasKey calls inTree
removeAny calls removeDeepestFromTree

Important Restrictions:

  1. Use KCompare's operations - use KCompare's areEqual operation for comparing two Key values for being equal and areOrdered for determining <= between two Key values. These are call back operations to client supplied code. Only the client, who supplied the Key and Value data types knows how to implement these operations that compare two Key values.
  2. BST - The supplied insertInTree operation will build a BST. All other operations must take advantage of this BST property and maintain the BST property
  3. No cross calling - None of Map's member functions are permitted to make calls to any of the other public Map member functions, i.e., no cross calling
  4. No iteration - None of Map's private helper member functions are permitted to use iteration, they must use recursion
  5. Use Design by Contract - No defensive programming of Map's operations.
  6. Making copies - Move (transfer) data, do not unnecessarily copy it. The only operation where a call to BinaryTree's assignment operator is permitted is in the Map3's operator =. Do not call operator = to make a copy of a binary tree for any other reason. Do not call operator = to make a copy of a variable of type K or V in any implementation of the other Map3 member functions.
    If you think you need to call operator = to make a copy in any Map operation (other than Map's operator =), then rethink your algorithm until you figure out a way to not make a copy.
  7. Binary searchÂ -When searching the BST for a specific Key value, your search must take advantage BST property and navigate to the searched for Key. No brute force traversing the tree to find a specific Key
