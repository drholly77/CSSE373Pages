2. L6 Statement of Work

2.1 Primary Objectives

The primary objective of this lab is to provide you experience working with:

  • Layering an operation (e.g., InjectCapability) on another already existing component (e.g., Queue)
  • Adding new functionality to an already existing component
  • The inheritance syntax used by C++ when layering a new operation
  • C++ syntax used by the client program when adding additional capabilities to an already existing component

2.2 Specifics

Steps:

  1. Start Visual Studio
  2. Open project L6
  3. Find and open L6.cpp
  4. Follow the four TODO's found near the top of L6.cpp

Hints:

  • AppendCapabilitiy1.hpp contains comments to explain the C++ syntax and how inheritance is working
  • The Reference Material section above has a link to a document containing an implementation for an inject operation

Important Restrictions:

  1. Use Design by Contract - No defensive programming for any of these operations
  2. Write elegant client code - Eliminate code duplication, have no unused variables, leverage Queue's public operations, etc.
  3. About Making Copies - There should be no reason to make copies of any Queue or variable of type T, so don't
