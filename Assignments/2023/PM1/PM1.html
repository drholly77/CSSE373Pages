1. Setup

1.1 Get the PM1Starter Download

  • Download PM1Starter.zip to your machine - to be used in subsequent steps
  

1.2 Copy the UnlabeledGraphDriver Folder from the PM1Starter.zip

  • Inside PM1Starter.zip is a project named UnlabeledGraphDriver
    • Copy the UnlabeledGraphDriver folder to your csse373devenv202330-yourID folder as shown in the screen shot to the right
    • Once you have copied this folder you can use VS2022 to open the UnlabeledGraphDriver project by double clicking on GraphDriver.sln

1.3 About Project UnlabeledGraphDriver

  • It contains a Visual Studio 2022 command line project
  • It can be opened with VS2022 by double clicking on the file GraphDriver.sln
  • It will build correctly if you have followed the instructions in 1.1 - 1.2 (above)
  • UnlabeledGraphDriver contains a command interpreter for UnlabeledGraphDriver component which is in your repo's components folder

2. PM1 Statement of Work

  • Implement UnlabeledUndirectedGraphChecking1
  • It must be:
    • implemented as shown in the Checking Component Construction instructional materials (see References in Section 0 above)
    • stored in the folder: components\include\UnlabeledUndirectedGraph
    • a checking component for the class UnlabeledUndirectedGraph1
      -- which is in the file UnlabeledUndirectedGraph1.hpp also found in the folder: components\include\UnlabeledUndirectedGraph
      -- here is a UML diagram for UnlabeledUndirectedGraph

3. Testing UnlabeledUndirectedGraphChecking1

  • Use the Graph Driver provided in PM1starter.zip to test your checking component
  • Note:
    • The Graph Driver is implemented so as to stop a keyboard user from violating some (possibly all) of UnlabeledUndirectedGraph1's operation's preconditions
    • You must slightly alter Graph Driver's code so that it allows the keyboard user to deliberately violate a precondition - you might consider creating an entire new project called NakedGraphDriver, where this driver has had all protection stripped from it, allowing a malicious keyboard user to execute commands that violate the UnlabeledUndirectedGraph1's operation's preconditions
    • This altered version will allow you to test from the keyboard your implementation of UnlabeledUndirectedGraphChecking1
  • The screen shot to the right shows how to start up the debugger in VisualStudio

4. Grading of PM1

  • Your UnlabeledUndirectedGraphChecking1.hpp will be compiled/built/tested in Debug Mode with Dr. Holly's private set of tests or an altered Graph Driver
  • These tests will act as defective clients of UnlabeledUndirectedGraph1 in the sense that they will violate the non-trivial preconditions of each UnlabeledUndirectedGraph1
  • The expectation during grading is: control will be transferred to the Visual Studio debugger as shown above in the two screenshots.
  • Zero points will be earned if your submission fails to build

5. Submitting the Assignment for Grading

  • Follow VS2022 instructions for committing and pushing your completed UnlabeledUndirectedGraphChecking1.hpp project to your GitHub Education CSSE373 repo
  • Use the commit message "PM1, ready for grading"
