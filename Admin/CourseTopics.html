CSSE373 Topics for Spring 2019

Module related:

  • Design by contract which assigns responsibility for the various parts of the contract
  • Common math theories used for a horizontal component base, e.g., string, integer, function, set, etc.
  • Use math theories to model the abstract type for a module, e.g., a math function to model a Map
  • Minimalist approach for selecting a module’s primary operations
  • Secondary operations
  • Multiple implementations for an abstraction
  • Layering one module on other modules utilizing internal contracts such as abstraction functions and module invariants

Logic and proof related:

  • Predicate logic
  • Logical implication, as well as logical and and logical or, etc.
  • Quantification
  • Direct proofs
  • Trivial and Vacuous proofs
  • Proof of recursive operations
  • Loop invariant proofs

Code:

  • Write programs using C++ software components whose design is informed by concepts in Formal Methods
  • Write programs using RESOLVE and Clemson's Web IDE
  • Modular reasoning and how abstraction leaks make reasoning not modular and harder
  • Conventions for reducing/eliminating abstraction leaks in components written in C++

  • Implement some components that have seen in data structures class, e.g., Map using hashing, Map using AVL/BST, Sequence using AVL/BT

Compiler related:

  • Know the functioning of a verifying compiler at a high level.
  • Learn about basics of a traditional compiler
  • Learn that the math theories need to be encoded so that they can be fed into the compiler
  • Learn what a verification condition is
  • Learn a bit of the process of VC generation for a small module and/or operation
  • Work with several examples of loop invariants, e.g., searching, sorting, an array, as well as loop invariants where the math model is a string, e.g., on Queues, Stacks, Lists

Testing:

  • Unit tests developed from a module’s spec, often referred to as specification-based testing
