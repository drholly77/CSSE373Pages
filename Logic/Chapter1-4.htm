1.4 Predicates and Quantifiers

Propositional logic cannot adequately express all mathematics and natural language statements.

Example

"2 > 3"    is valid proposition.

"x > 3"    is not valid proposition, x is undefined.

Predicates

Example

"x > 3" has two parts

  1. variable: "x"
    is the subject of the statement
  2. predicate: "> 3"
    refers to a property that the subject can have

P( x ) denotes the statement "x > 3 "
After x has been assigned a value, P(x) becomes a proposition, i.e., is either True or False

P(4) is True        "4 > 3"

P(2) is False       "2 > 3" 

Quantifiers

Definition - Universal Quantification
Universal quantification of P( x ) is the statement:

"P( x ) for all values of x in the domain."

∀x P( x )

denotes universal quantification of P( x )

Example

P(x) is "x > x - 1"

P(2) is "2 > 2 - 1"

Assume the domain: x ∈ { 0, 1, 2 }

P(0) is "0 > 0 - 1"

P(1) is "1 > 1 - 1"

P(2) is "2 > 2 - 1"

∀x P(x) = P(0) ∧ P(1) ∧ P(2) = True

Definition - Existential Quantification
Existential quantification of P(x) is the statement:

"There exists an element x in the domain such that P(x) is True."

∃x P(x)

denotes existential quantification of P(x)

Example

∃x P(x) is True

when at least one P(x) is True

P(x) is "(x * x) > x"

Assume the domain: x ∈ {-2, -1, 0}

Show disjunction of P(x) is True for at least one x ∈ {-2, -1, 0}:

P(-2) ∨ P(-1) ∨ P(0)

P(x) is             "(x * x) > x"

P(-2) is True    "4 > -2"

P(-1) is True    "1 > -1"

P(0) is False    "0 > 0"

P(-2) ∨ P(-1) ∨ P(0) = (True ∨ True ∨ False) which is True

∃x P(x) is True

when at least one P(x) is True

Constraints on Quantifier Variables

When variable x is of type T write:

∀ x:T (P(x))

∃ x:T (P(x))

In a ∀, where x is constrained to specific values write:

∀ x:T (P(x) → Q(x))

where P(x) defines the constraints on x's values

In a ∃, where x is constrained to specific values write:

∃ x:T (P(x) ∧ Q(x))

where P(x) defines the constraints on x's values

Examples

∀ x:Integer (P(x))

∀ x:Integer ((0 ≤ x < N) → Q(x))

∃ y:Integer (P(y))

∃ y:Integer ((0 ≤ y < N) ∧ Q(y))

Precedence of Quantifiers

∃ and ∀ are higher than all operators.

Example

∃x P(x) ∧ Q(x)

means (∃x P(x) ) ∧ Q(x)

not ∃x ( P(x) ∧ Q(x) )

you need parenthesis if you want the quantifier to apply to both P(x) and Q(x)

 

Logical Equivalences Involving Quantifiers

Negating Quantified Expressions
~∀x P( x )   ≡  ∃x ~P( x )


~∃x Q( x )   ≡  ∀x ~Q( x )

 

Example

"Everyone in CSSE373 has a cell phone."

P is "has a cell phone"

x is in domain "CSSE373".

P(x) is x in domain "CSSE373 has a cell phone".

∀x P(x) is "Everyone in CSSE373 has a cell phone."
 

~∀x P(x)  ≡  ∃x ~P(x)

the negation would be:

~∀x P(x) "Not everyone in CSSE373 has a cell phone."

∃x ~P(x) "There exists at least one person in CSSE373 without a cell phone."

Translating from English into Logical Expressions

Example

x is in domain of "all people."

S(x) is "x is a student in CSSE230".

M(x) is "x has visited Mexico.

"Some student in CSSE230 has visited Mexico."

"There exists a student x in CSSE230 such that x has visited Mexico"

∃x (S(x) ∧ M(x))

 

Note, cannot use S(a) → M(a)

"If a is student in CSSE230 then a has visited Mexico"

because the conditional S(a) → M(a) is True when S(a) is False:

∃x (S(x) ∧ M(x))  ≠  ∃x (S(x) → M(x))

∃x (S(x) → M(x)) would be True for everyone not in CSSE230.
